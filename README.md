# A 股交互式交易策略（C++）

一个用于算法交易竞赛的 C++ 程序。

## 快速开始

### 编译
```bash
g++ -std=c++17 -O2 -Wall -o trading_agent src/main.cpp
```

### 运行
```bash
./trading_agent < data/sample_input.txt
```

**注意**：该程序是一个**交互程序**，需要与裁判系统（Judge）实时通信，直接重定向输入无法完整模拟交互过程。本地调试时建议配合模拟 Judge 脚本使用（见下文）。

## 交互协议

- 程序从标准输入读取**INIT** 参数和每日行情。

- 每轮输出委托后，输出 `DONE` 并 flush。
- 最后一交易日结束后等待 `FINISH <资产>` 并退出。

- 所有金额计算四舍五入至分，涨停日拒绝买入，停牌委托直接丢弃。

## 策略概述（`src/main.cpp`）

- **动量选股**：计算过去 5 个交易日收益率，选择收益最高且为正的 5 只股票。
- **等权重配置**：卖出非持仓，将目标持仓调整至等市值。
- **资金管理**：买入时预估成本，避免生成必然被拒的委托。
- 该策略仅作为示例，你可以替换 `generate_orders` 函数来实现自己的策略。

## 自定义策略

1. 修改 `generate_orders(int day)` 函数。
2. 可利用 `CLOSES`、`OPENS`、`holdings`、`cash` 等全局状态。
3. 注意保持输出的委托数量不超过 `K_MAX`。

## 注意事项

- 所有价格和金额保留两位小数，四舍五入。
- 卖出会扣除印花税（`BETA`）和佣金，买入只扣佣金。
- 涨停板判断：当日开盘价 >= 前收盘价 * 1.10 则无法买入。
- 停牌（`open == 0`）委托不成交。
